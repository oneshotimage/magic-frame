# AI 影像写真馆 MVP - API 模型可行性测试设计

## 1. 测试目标

验证 KL API（`kl-api.apifox.cn`）聚合的以下模型是否支撑《AI影像特效小程序 PRD（MVP正式版）》的核心需求：

- Liblib API
- 即梦/豆包图像模型
- GPT Image 2（以 KL 文档实际 model id 为准）
- Google Nano Banana / Gemini Flash Image（以 KL 文档实际 model id 为准）

MVP 核心任务不是文生图，而是「上传一张清晰人像，稳定生成四种固定风格写真」。测试必须优先覆盖图生图、人物一致性、出图稳定性、耗时、失败率、合规拦截和单次成本。

## 2. PRD 验收约束

| 维度 | PRD 要求 | 测试判定 |
| --- | --- | --- |
| 生成方式 | 用户上传图片，一次返回 4 张风格图 | 每个模型分别测试四风格图生图能力 |
| 平均耗时 | AI 生成平均 4-10 秒 | P50 <= 10s，P95 <= 15s |
| 最大等待 | 15 秒超时 | 超过 15s 记为 MVP 不达标 |
| 成功率 | 生成成功率 >= 95% | 按有效请求统计成功率 |
| 计费 | 成功返回图片才扣费 | 记录失败/超时是否计费 |
| 内容安全 | 色情、暴力、政治敏感、人脸异常拦截 | 验证模型/API 错误码与业务兜底 |
| 输出 | 无水印，原图分辨率保存 | 检查水印、尺寸、清晰度 |

## 3. 测试样本集

建议准备 30 张授权人像图，全部压缩到 2MB 以内，统一保留原图副本。

| 样本组 | 数量 | 目的 |
| --- | ---: | --- |
| 成人正面清晰单人 | 8 | 核心成功路径 |
| 女性/男性/不同年龄 | 6 | 检查性别、年龄泛化 |
| 儿童亲子照片 | 4 | 验证亲子用户场景，但注意合规 |
| 弱光/逆光/普通自拍 | 4 | 检查真实用户输入容错 |
| 轻微遮挡/戴眼镜/帽子 | 4 | 检查五官保持 |
| 多人脸/非人像/模糊图 | 4 | 验证前置拦截与失败不扣费 |

最小首轮可以先用 8 张样本跑 smoke test：5 张有效人像，3 张异常输入。

## 4. 四种风格 Prompt

所有模型使用同一套中文 prompt，必要时追加英文补充，避免模型之间因提示词差异造成误判。

### 3D 皮克斯卡通

保留上传照片中人物的身份特征、脸型、发型、表情和姿态，将人物重绘为高质量 3D 动画电影角色。风格为柔和立体、干净皮肤、明亮眼睛、精致布光、温暖色彩，适合微信头像和亲子写真。不要改变人物性别、年龄、发型主体和服装主体，不要增加多余人物，不要出现文字、水印、畸形五官。

### 高级写实插画

保留上传照片中人物的身份特征、脸型、五官比例、发型、表情和姿态，将人物重绘为高级写实插画写真。画面需要自然肤色、电影级光影、精致细节、干净背景、轻微艺术化但接近真人，适合朋友圈精美配图。不要过度磨皮，不要改变人物身份，不要出现文字、水印、畸形手指或五官。

### 文艺手绘质感

保留上传照片中人物的身份特征、脸型、发型、表情和姿态，将人物重绘为温柔文艺手绘写真。风格为柔和线条、纸张纹理、淡雅配色、治愈氛围、轻插画质感，适合头像和日常分享。不要改变人物身份、年龄、性别，不要增加文字、水印或多余人物。

### 潮流涂鸦漫画

保留上传照片中人物的身份特征、脸型、发型、表情和姿态，将人物重绘为潮流街头漫画风格。风格为清晰轮廓、漫画分镜质感、适度涂鸦元素、强对比但不脏乱，适合年轻用户头像和社交分享。不要改变人物身份，不要生成品牌 logo、文字、水印、畸形五官或多余人物。

## 5. API 调用设计

### 环境变量

```bash
export KL_API_KEY="替换为 KL API Key"
export KL_BASE_URL="以 kl-api.apifox.cn 文档中的接口地址为准"
```

### 调用形态

优先选择支持「图片输入 + 文本 prompt + 图片输出」的图生图/图片编辑接口。若某个模型只支持文生图，应判定为不适合 MVP 首发主链路，只可作为营销素材生成或兜底实验。

每次调用记录：

- `request_id`
- `model`
- `style`
- `sample_id`
- `input_size_kb`
- `start_time`
- `first_response_time_ms`
- `finish_time_ms`
- `status`
- `error_code`
- `charged`
- `output_url_or_file`
- `output_width`
- `output_height`
- `watermark_detected`

### 通用请求模板

具体 endpoint 和字段名以 KL Apifox 文档为准。测试代码里需要把四个模型封装为同一接口：

```json
{
  "model": "MODEL_ID_FROM_KL_DOCS",
  "image": "input image url or base64",
  "prompt": "style prompt",
  "size": "1024x1024",
  "response_format": "url",
  "n": 1
}
```

如果 KL 使用异步任务接口，测试流程固定为：

1. `submit task`
2. 每 1 秒轮询一次
3. 15 秒未完成即业务超时
4. 30 秒内继续后台观察真实完成时间
5. 记录 15 秒超时率和最终成功率

## 7. 人工评分标准

每张输出图按 1-5 分评分，最终按权重汇总。

| 指标 | 权重 | 5 分标准 |
| --- | ---: | --- |
| 人物一致性 | 30% | 一眼能认出同一个人，年龄/性别/脸型基本不变 |
| 风格命中 | 20% | 明确符合目标风格，不串风格 |
| 美观度 | 20% | 可直接保存/分享，构图、色彩、质感高级 |
| 五官与肢体正确性 | 15% | 无明显畸形、斜眼、牙齿/手指异常 |
| 小程序适配 | 10% | 正方形头像/写真可用，无裁切问题 |
| 合规与水印 | 5% | 无敏感内容、无水印、无文字污染 |

模型上线阈值：

- 综合评分 >= 4.2：可作为主模型
- 3.8-4.2：可作为备选/部分风格模型
- < 3.8：不建议进入 MVP 主链路

## 8. 自动化指标

| 指标 | 计算方式 | MVP 阈值 |
| --- | --- | --- |
| 成功率 | 成功返回图片 / 有效请求 | >= 95% |
| 15 秒内完成率 | 15 秒内成功 / 有效请求 | >= 90% |
| P50 延迟 | 成功请求耗时中位数 | <= 10s |
| P95 延迟 | 成功请求 95 分位 | <= 15s |
| 超时率 | 15 秒未完成 / 有效请求 | <= 10% |
| 单套成本 | 4 张风格图总成本 | 需低于业务毛利要求 |
| 异常误扣费 | 失败/拦截但扣费次数 | 必须为 0 |

## 9. 推荐判定逻辑

测试结束后不要只选一个“总分最高”的模型，应按风格拆分：

- 3D 卡通：优先选人物一致性强、可爱但不低龄化过度的模型
- 写实插画：优先选五官稳定、皮肤自然、不过度网红脸的模型
- 文艺手绘：优先选色彩和线条稳定的模型
- 涂鸦漫画：优先选风格命中高、但不乱加文字和 logo 的模型

如果某个模型在所有风格都稳定，MVP 可以采用单模型四风格，后端更简单。如果各模型各有优势，可以采用「按风格路由」：

| 风格 | 主模型 | 兜底模型 |
| --- | --- | --- |
| 3D 皮克斯卡通 | 待测试 | 待测试 |
| 高级写实插画 | 待测试 | 待测试 |
| 文艺手绘质感 | 待测试 | 待测试 |
| 潮流涂鸦漫画 | 待测试 | 待测试 |

## 10. 风险点

1. GPT Image 2、Nano Banana 等模型可能更擅长高质量编辑，但延迟或单价未必适合“一次生成 4 张”的小程序次数包。
2. Liblib/即梦可能在中文审美和爆款风格上更贴近国内用户，但要重点测水印、失败重试、图片保存分辨率和内容安全错误码。
3. 一次请求生成 4 张如果接口不支持，需要并发 4 个风格任务，后端必须实现任务聚合、局部失败兜底和超时返回。
4. PRD 的 15 秒上限偏紧。若完整 4 图 P95 超过 15 秒，建议产品改为“先展示已完成图片，其余继续生成”或“生成完成后服务通知”。
5. 儿童照片涉及更高合规要求，MVP 应避免营销中强调儿童换装、性感化、职业制服等敏感方向。

## 11. 最终输出物

测试完成后输出：

- `model_results.csv`：每次调用的延迟、状态、费用、输出路径
- `score_sheet.csv`：人工评分明细
- `outputs/{model}/{style}/{sample_id}.png`：生成结果归档
- `summary.md`：主模型推荐、兜底模型、成本估算、PRD 是否需要调整
