# AI 影像写真馆规格说明

## 目标

构建一个可上线微信小程序的 AI 写真生成产品，用户可以登录、上传本人或已授权人像、选择风格、创建生成任务、查看结果、保存图片、分享海报、管理作品、购买额度，并由后台查看任务、订单、用户、反馈和图片资产。

## 用户角色

- 游客用户：可浏览首页、指南、协议、隐私、常见问题；触发生成、作品、充值等动作时再引导登录。
- 登录用户：可上传图片、生成写真、保存作品、分享、查看作品集、管理个人信息、购买额度。
- 管理员：可通过后台查看运行状态、用户、任务、订单、反馈、图片资产，并处理异常任务。

## 核心流程

1. 用户进入首页。
2. 用户点击制作，必要时完成微信登录。
3. 用户选择风格。
4. 用户上传正面照片。
5. 小程序提交图片校验。
6. 用户确认授权与合规声明。
7. 后端创建生成任务并调用 KL API。
8. 小程序轮询任务状态。
9. 任务完成后展示 HTTP 图片 URL。
10. 用户保存图片、生成分享海报或进入作品集。
11. 管理员可在后台查看任务详情和错误摘要。

## 小程序规格

### 页面

- 首页：展示产品入口、生成入口、自定义 tabbar。
- 使用指南：说明生成步骤和上传要求。
- 登录页：微信一键登录，失败时展示明确错误。
- 风格选择页：选择一种或多种写真风格。
- 上传页：相册选择、相机拍摄、预览、继续生成。
- 确认页：确认照片和重新选择。
- 授权确认页：确认人像授权、内容合规和生成风险。
- 生成中页：展示进度、调用模式、模型、接口、错误摘要。
- 结果页：展示生成图片、失败项、保存、分享、重试。
- 预览页：大图预览、保存、分享。
- 保存成功页：跳转作品集、首页或分享海报。
- 分享海报页：展示生成海报并支持分享。
- 购买页：套餐、订单、模拟或正式支付入口。
- 广告奖励页：测试广告奖励流程。
- 个人中心：登录态、额度、菜单、充值入口、自定义 tabbar。
- 作品集：历史作品、筛选、自定义 tabbar。
- 订单页：展示充值和支付状态。
- FAQ、用户协议、隐私政策、反馈页：基础支持页面。

### 组件

- `nav-bar`：统一自定义导航栏；必须读取状态栏高度，导航栏位于安全区以下。
- `app-button`：统一主按钮；支持 loading、disabled、open-type、variant。
- `app-tabbar`：统一底部导航；首页、作品集、我的页面使用。

### UI 安全区

- 顶部：自定义导航栏使用 `wx.getSystemInfoSync().statusBarHeight`。
- 固定导航栏页面内容：顶部留白使用 `env(safe-area-inset-top)` 叠加设计间距。
- 底部：固定按钮区和 tabbar 使用 `env(safe-area-inset-bottom)`。

## 后端规格

### API

后端应提供以下能力：

- Auth：微信登录、刷新 token、退出登录。
- User：资料读取、更新、注销。
- Credits：额度读取、日志、消耗、广告奖励。
- Upload：图片上传、图片校验。
- Generation：创建任务、查询任务、重试、取消、历史记录。
- Packages / Orders / Payment：套餐、订单、支付通知、对账。
- Share：生成分享海报、分享奖励。
- Feedback：提交反馈。
- Admin：用户、任务、订单、反馈、图片资产、运行配置。

### 生成任务

任务状态：

- `pending`
- `processing`
- `succeeded`
- `failed`
- `canceled`

任务结果要求：

- 成功图片必须返回 HTTP URL。
- KL 返回 `b64_json` 或远程图片 URL 时，后端必须转存为图片资产后返回 `/assets/generated/{assetId}.png` 或 COS 对象存储 URL。
- 失败图片必须记录错误消息、provider、model、endpoint、HTTP 状态和对象存储错误摘要。
- 任务响应必须包含 `startedAt`、`completedAt`、`elapsedMs` 等耗时字段；生成中页离开后后端任务继续执行，除非用户显式取消任务。

### 存储

- 本地开发：未配置数据库时可使用 SQLite 和本地对象目录。
- 云托管：优先使用 MySQL 和 COS，正式环境设置 `OBJECT_STORAGE_STRICT=1`。
- 图片资产：上传图、生成图、分享海报应写入 COS，并返回可被小程序真机访问的 HTTP/HTTPS URL。

## 管理后台规格

后台路径：

```text
/admin
```

后台 API：

```text
/admin/api/*
```

后台能力：

- 运行配置和健康状态。
- 用户列表、用户详情、额度调整。
- 任务列表、任务详情、结果图预览、错误摘要、重试、取消。
- 订单列表、关闭待支付订单。
- 反馈列表。
- 图片资产列表和预览。

## 部署规格

FastAPI 云托管部署：

- 构建上下文：仓库根目录。
- Dockerfile：`Dockerfile`。
- 启动命令：`uvicorn backend.main:app --host 0.0.0.0 --port ${PORT:-80}`。
- 健康检查：`GET /health`。

正式环境必须配置：

- `PUBLIC_BASE_URL`
- `KL_API_TOKEN`
- `KL_IMAGE_MODEL`
- `KL_IMAGE_ENDPOINT`
- `KL_IMAGE_SIZE`
- `AI_MOCK_GENERATION=0`
- `AI_UNLIMITED_CREDITS=0`
- 数据库环境变量
- COS 环境变量

## 非功能要求

- 真机图片可加载。
- 分享页必须展示生成图或海报图。
- 登录失败不能无限转圈。
- 未登录不自动跳回登录页。
- 生成失败要可诊断。
- 后台和小程序展示同一套生成图片 URL。
- 关键流程本地可测试，云托管可健康检查。
