# 妙影工坊 MagicFrame 任务拆分

## P0 上线阻塞

1. 登录稳定性
   - 修复真机微信一键登录长时间 loading。
   - 明确处理 `-109`、code 获取失败、后端换 token 失败。
   - 未登录时只在用户触发受保护操作后引导登录。

2. 图片展示链路
   - 确认生成图统一返回 HTTP URL。
   - 确认小程序、分享页、后台管理均使用同一图片 URL。
   - 确认 `PUBLIC_BASE_URL` 在本地、云托管、COS 场景下正确。
   - 禁止向小程序结果页直接返回 base64 `data:` URL。

3. 分享链路
   - 分享海报页必须优先展示后端海报图。
   - 海报生成失败时回退展示原生成图。
   - 分享参数中保留 taskId、imageId 或 posterId。

4. 云托管部署
   - 使用 FastAPI Dockerfile 部署到微信云托管。
   - 配置 MySQL、COS、KL API、管理员账号。
   - 部署后验证 `/health`、`/config/runtime`、`/admin`。

5. 小程序 UI 统一
   - 全页面导航栏使用 `nav-bar`。
   - 主按钮使用 `app-button`。
   - 首页、作品集、我的使用 `app-tabbar`。
   - 所有自定义导航页面处理顶部状态栏安全区。

## P1 核心体验

1. 生成任务体验
   - 生成中页展示可理解的进度状态。
   - 结果页展示成功图和失败项。
   - 支持失败项重试。
   - 任务取消、超时、失败状态文案清晰。

2. 作品集
   - 支持按风格筛选。
   - 支持从作品进入预览。
   - 空状态引导用户去生成。

3. 购买和额度
   - 套餐展示和选中态。
   - 下单、支付、订单查询。
   - 支付成功后额度增加。
   - 测试模式和正式模式区分明确。

4. 管理后台
   - 任务详情展示 provider 调用信息。
   - 图片资产可预览。
   - 用户额度可调整。
   - 反馈可查看和处理。

## P2 完善项

1. 内容安全
   - 上传校验扩展为真实内容审核。
   - 生成结果审核。
   - 敏感失败原因对用户脱敏，对后台保留原始摘要。

2. 可观测性
   - 记录任务耗时、成功率、失败率。
   - 记录 KL API 请求模型、endpoint、HTTP 状态。
   - 管理后台增加趋势图或导出。

3. 成本控制
   - 单用户并发任务限制。
   - 失败不扣费。
   - 重试策略和超时策略。

4. 安全
   - 管理员密码强度和登录保护。
   - Token 过期和刷新策略。
   - 图片 URL 防盗链或短期签名策略。

## 建议执行顺序

1. 先跑通 FastAPI 本地服务、小程序真机登录和图片展示。
2. 再完成云托管环境变量、MySQL、COS。
3. 然后验证生成、结果、分享、作品集、后台图片预览。
4. 最后处理支付、额度、广告奖励和运营后台增强。
